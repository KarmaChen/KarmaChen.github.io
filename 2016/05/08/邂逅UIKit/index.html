UIKit

iOS应用程序都基于UIKit，你不能是应用程序脱离这个框架。UIKit提供了在屏幕上绘制的机制，捕获事件，和创建通用用户界面元素。

UIGestureRecognizer手势

UIGestureRecognizer的简介:

UIGestureRecognizer类是一个抽象类，不能直接使用，具体使用过程中需要使用其子类。UIGestureRecognizer一共包括6个子类，分别对应6种手势，即：
UITapGestureRecognizer：点击
UIPinchGestureRecognizer：捏合
UIRotationGestureRecognizer：旋转
UISwipeGestureRecognizer：滑动
UIPanGestureRecognizer：拖动，其又包含另外一个子类：UIScreenEdgePanGestureRecognizer
UILongPressGestureRecognizer：长按

创建一个UIGestureRecognizer子类对象需要有4个步骤：

1.创建一个UIView类的对象；
2.创建一个手势类对象，并使用- initWithTarget:action:方法进行对象初始化，设置当手势事件发生时，调用的方法；
example:

3.为UIView对象添加手势，使用 addGestureRecognizer：方法

4.实现手势事件发生时所调用的方法。

滑动的方向:
UISwipeGestureRecognizerDirectionRight：向右，默认；
UISwipeGestureRecognizerDirectionLeft：向左；
UISwipeGestureRecognizerDirectionUp：向上；
UISwipeGestureRecognizerDirectionDown：向下。

2.长按手势一定需要和UIGestureRecognizer类的state属性进行配合使用，即通过不同状态state下，实现不同的功能。UIGestureRecognizer类的state属性中，常见的有：
UIGestureRecognizerStateBegan：开始手势事件
UIGestureRecognizerStateEnded：结束手势事件
UIGestureRecognizerStateChanged：手势位置发生变化
UIGestureRecognizerStateFailed：无法识别的手势

UITableView

UITableView的数据源协议(UITableViewDelegate,UITableViewDatasource)有3个常用方法，其中2个是必选方法.

###必选
返回每个段（seciton）中有多少个单元格；

根据indexPath属性，返回该单元格的具体样式和显示内容；

###可选
返回整个tableview有多少个段（section），如果不实现该方法，默认为1个段。

###UITableView的cell点击事件的代理方法
单元格Cell，都可以监听用户的点击动作，并通过代理方法来进行一些逻辑处理。常见的操作有：选中某一个单元格后，能够跳转到另外一个控制器，显示详细信息.

UICollectionView

简介:集合视图提供网状的排列,集合视图的核心在于显示数据和布局方式.
视图的组成:单元格cell+补充视图Supplementary View+Decoration View

布局分两种:
1.custom Layout : 自定义布局
2.flow Layout : 线性布局(UICollectionViewFlowLayout：Flow Layout是一个Cells的线性布局方案，并具有页面和页脚)
